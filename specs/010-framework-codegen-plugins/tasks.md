## Phase 3: protoc-gen-storage

- [X] T004 [C] Implement `cmd/protoc-gen-storage/main.go`.
  - `protogen.Options{}.Run(...)` entry point.
  - `generateFile(gen, f)`: iterate messages in the file.
  - For each message: collect scalar fields (skip repeated, message-type, map, oneof).
    Identify the `id` field (name == "id", type string); fall back to first string field.
  - Output package: `<pbPkgName>storage` (e.g. `widgetsv1storage`).
    Output file: `<f.GeneratedFilenamePrefix>.storage.go`.
  - Content:
    - `type <Message>Model struct { ID string \`gorm:"primaryKey;type:varchar(36)"\`; <field>
      <type> \`gorm:"column:<snake>"\`; ...; ETag string \`gorm:"column:etag"\`; CreatedAt
      time.Time; UpdatedAt time.Time; DeletedAt gorm.DeletedAt \`gorm:"index"\` }`
    - `func toModel(p *pb.<Message>) *<Message>Model { ... }`
    - `func fromModel(m *<Message>Model) *pb.<Message> { ... }`
    - `type <Message>Repository struct { db *gorm.DB }`
    - `func New<Message>Repository(db *gorm.DB) *<Message>Repository { ... }`
    - Five CRUD methods (Get/List/Create/Update/Delete).
    - `var _ persistence.Repository[*pb.<Message>, string] = (*<Message>Repository)(nil)`
    - TODO comments for skipped fields.
  - Imports: `context`, `encoding/base64`, `fmt`, `strconv`, `time`,
    `gorm.io/gorm` (as literal import path string, not a Go dep of the plugin binary),
    `github.com/infobloxopen/devedge-sdk/persistence`.
  - File carries `// Code generated by protoc-gen-storage. DO NOT EDIT.` header.
  - Skip files with no messages.
  Unit tests from T002 must go green.
